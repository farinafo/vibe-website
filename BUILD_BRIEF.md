# Personal Portfolio Website Build Brief

## 1. Project Goal
Build a personal portfolio website for Chen Fan, aimed at job applications in AI product, product growth, and international user insight related roles.

The site should help recruiters, hiring managers, and collaborators quickly understand:
- who I am
- what I work on
- what projects best represent me
- why I am a strong fit for AI product / growth / cross-cultural roles

This is not a generic personal homepage.
It should feel like a polished portfolio site with strong project storytelling.

---

## 2. Target Audience
Primary audience:
- recruiters
- hiring managers
- interviewers

Secondary audience:
- potential collaborators
- people interested in my work

The site should optimize for:
- fast understanding
- clean project presentation
- credibility
- strong personal positioning

---

## 3. Core Positioning

Positioning:

---

## 4. Visual Direction
The visual style should be strongly inspired by:
https://www.edoardolunardi.dev/?ref=siteinspire

Important:
- The site should be inspired by its visual language, layout rhythm, minimalism, typography, and motion feeling
- Do NOT copy content, branding, or exact assets
- The final website should still reflect my own identity and portfolio structure

Visual keywords:
- minimal
- elegant
- editorial
- premium
- black and white base
- refined spacing
- modern typography
- subtle but high-quality motion
- clean project-first portfolio feel

Motion keywords:
- smooth page transitions
- subtle hover motion
- refined reveal animations
- calm scrolling experience
- premium interaction feedback
- not flashy, not overly playful

Overall mood:
A polished portfolio website that feels design-conscious, modern, and internationally credible.

---

## 5. Tech Stack
Use:
- Next.js
- TypeScript
- Tailwind CSS

Preferred additions:
- Framer Motion for interactions and transitions
- simple reusable layout and section components
- responsive design by default
- easy to deploy on Vercel

Do not add:
- CMS
- database
- login
- admin panel
- heavy backend
- unnecessary dependencies

This should be a clean static portfolio site.

## 13. Design Rules
- Chinese-first design
- minimal but not empty
- no overly cute or playful style
- no bright colors
- typography and spacing should carry the design
- motion should be premium and restrained
- use generous whitespace
- emphasize readability and layout rhythm

---

## 14. MVP Scope
Build first:
- full site structure
- all pages
- responsive layout
- reusable components
- smooth interactions
- placeholder images where needed

Do not build yet:
- English version
- blog system
- CMS
- dark/light toggle unless easy to include
- analytics integration
- advanced SEO system
- custom domain config

---

## 15. Deliverable Expectations
I want a real, production-ready portfolio website that:
- looks polished
- feels close in style quality to the reference site
- presents my projects clearly
- is easy to continue iterating later
- is deployable to Vercel
